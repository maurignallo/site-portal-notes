# site-portal-notes

本仓库用于归档和发布多个独立的 HTML 页面。这些页面内容相互独立，不针对特定网站或域名，主要作为信息整理与展示的静态资源集合。

## 项目简介

该仓库旨在提供一个轻量级的 HTML 页面发布与归档空间。所有页面均为纯前端实现，不依赖后端服务，适合用于保存静态文档、工具页面、信息面板等。仓库本身不提供搜索、用户系统或动态数据功能，仅作为 HTML 文件的托管入口。

## 目录说明

```
.
├── index.html          # 主入口页面（可选）
├── pages/              # 存放所有独立 HTML 页面
│   ├── page-a.html
│   ├── page-b.html
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `index.html`：可选的主页，通常包含页面导航或简介。
- `pages/`：所有独立 HTML 页面均放置于此目录，每个页面可自包含样式与脚本，或引用 `assets/` 中的公共资源。
- `assets/`：存放页面间共享的样式表、JavaScript 库、图片等静态文件。

## 页面归档说明

- 每个 HTML 页面均为完整可独立运行的文档，无需依赖其他文件（除公共资源外）。
- 页面内容不包含任何外部链接、跟踪代码或广告元素。
- 归档仅保留最终发布的版本，不维护历史修改记录。
- 页面命名建议使用小写字母、数字与连字符，避免空格与特殊字符。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，若需引用公共资源，请使用相对路径（如 `../assets/css/style.css`）。
- 更新页面：直接替换或修改对应 HTML 文件内容。
- 删除页面：移除对应文件，并检查 `index.html` 中是否有相关链接。
- 建议每次提交附上简要说明，描述新增、修改或删除的内容。

## 使用方式

1. 克隆或下载本仓库。
2. 直接在本地浏览器中打开 `index.html` 或 `pages/` 下的任一 HTML 文件即可查看。
3. 若需发布到 GitHub Pages，请在仓库设置中启用 Pages 功能，并将发布源指向根目录。
